# Internacionalizacion (i18n)

El Template viene con infraestructura `gettext` activa para dos idiomas:

- `es` — Espanol (default cuando el browser no especifica)
- `en` — English

Catalogos en `locale/<code>/LC_MESSAGES/django.po`. Los `.mo` compilados
viven al lado.

## Como Django decide el idioma

`LocaleMiddleware` (en `settings.py`) selecciona el idioma del request:

1. URL prefix `/<lang>/...` si esta habilitado (no por ahora)
2. Cookie `django_language`
3. Header `Accept-Language` del browser
4. Default `LANGUAGE_CODE = "es-cl"` (fallback a `es`)

Si el browser pide `Accept-Language: en-US,en;q=0.9`, el Template
responde con los strings de `locale/en/LC_MESSAGES/django.mo`.

## Marcar strings nuevos

En Python:

```python
from django.utils.translation import gettext as _, gettext_lazy as __

# Strings que se evaluan en cada request (vistas, mensajes):
messages.success(request, _("Perfil actualizado."))

# Strings que se evaluan al import-time (campos de modelos, choices):
class User(models.Model):
    role = models.CharField(choices=[("public", __("Publico"))])
```

En templates Django:

```django
{% load i18n %}
<button>{% trans "Crear token" %}</button>
<p>{% blocktrans %}Bienvenido, {{ user }}.{% endblocktrans %}</p>
```

## Regenerar los catalogos

Despues de agregar/cambiar strings:

```bash
cd /opt/ameli-app-template-dev
.venv/bin/django-admin makemessages -l es -l en \
    --ignore=tests --ignore=.venv --ignore=docs --no-location
```

Editar los `.po` resultantes (uno por idioma), llenar `msgstr ""` con la
traduccion. Despues compilar:

```bash
.venv/bin/django-admin compilemessages
```

(O ejecutar `msgfmt locale/<lang>/LC_MESSAGES/django.po -o locale/<lang>/LC_MESSAGES/django.mo`
manualmente si Django no esta a mano.)

Los `.mo` son binarios pero **deben commitearse** al repo: production
no tiene `msgfmt` siempre disponible y queremos despliegues idempotentes.

## Cambiar el idioma manualmente

Para verificar que la traduccion funciona, podes forzar el idioma con
una cookie:

```
document.cookie = "django_language=en; path=/";
location.reload();
```

O via curl:

```bash
curl http://10.100.100.16:18080/profile/ \
    -H "Accept-Language: en" \
    -b cookies.txt
```

## Strings no marcados

No todos los strings del Template estan traducidos: el bloque inicial
cubre los mensajes mas visibles (auth, errores de validacion, throttle).
A medida que las apps heredadas (Starlink, Metro) necesiten otro idioma,
marcan sus propios strings con `_()` y regeneran sus catalogos. El
proceso de arriba aplica identico.

## Agregar un idioma nuevo

```python
# settings.py
LANGUAGES = [
    ("es", "Espanol"),
    ("en", "English"),
    ("pt", "Portugues"),  # nuevo
]
```

```bash
mkdir -p locale/pt/LC_MESSAGES
.venv/bin/django-admin makemessages -l pt --ignore=tests --no-location
# Editar locale/pt/LC_MESSAGES/django.po, llenar msgstr
.venv/bin/django-admin compilemessages
```
